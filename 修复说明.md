# 问题修复说明

## 问题分析

您遇到的两个问题已经得到解决：

### 问题 1：GitHub Actions 无权创建 Pull Request

**错误信息：**
```
pull request create failed: GraphQL: GitHub Actions is not permitted to create or approve pull requests (createPullRequest)
```

**原因：**
出于安全考虑，GitHub 默认不允许工作流使用 `GITHUB_TOKEN` 创建 Pull Request。这是为了防止恶意代码通过自动创建的 PR 触发其他工作流。

**解决方案：**

您需要选择以下两种方法之一：

#### 方法 1：在仓库设置中启用（推荐）

1. 打开您的 GitHub 仓库
2. 点击 **Settings**（设置）
3. 点击左侧 **Actions** → **General**
4. 滚动到 **Workflow permissions**（工作流权限）部分
5. 勾选：**"Allow GitHub Actions to create and approve pull requests"**（允许 GitHub Actions 创建和批准 Pull Request）
6. 点击 **Save**（保存）

#### 方法 2：使用个人访问令牌（PAT）

如果您不想给所有工作流授予创建 PR 的权限，可以使用 PAT：

1. 创建 Personal Access Token：
   - 访问 https://github.com/settings/tokens
   - 点击 **Generate new token** → **Generate new token (classic)**
   - 名称填写："Copilot Configs Sync"
   - 权限勾选：`repo`（完整的仓库控制权限）
   - 点击 **Generate token**
   - **复制生成的令牌**（稍后不会再显示）

2. 将令牌添加到仓库密钥：
   - 进入仓库 **Settings** → **Secrets and variables** → **Actions**
   - 点击 **New repository secret**
   - 名称：`PAT_TOKEN`
   - 值：粘贴刚才复制的令牌
   - 点击 **Add secret**

3. 更新工作流文件（`.github/workflows/sync-copilot-configs.yml`）：
   - 找到 "Create or update pull request" 步骤
   - 注释掉：`GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}`
   - 取消注释：`GH_TOKEN: ${{ secrets.PAT_TOKEN }}`

### 问题 2：Node.js 20 弃用警告

**警告信息：**
```
Warning: Node.js 20 is deprecated. The following actions target Node.js 20 but are being forced to run on Node.js 24: actions/checkout@v4.2.1
```

**原因：**
GitHub Actions 正在逐步淘汰 Node.js 20，转而使用 Node.js 24。旧版本的 actions 仍然使用 Node.js 20。

**解决方案：**

已经在工作流中修复：
- 将 `actions/checkout@v4.2.1` 升级到 `actions/checkout@v4`
- v4 版本会自动使用运行器支持的最新 Node.js 版本

下次运行工作流时，警告将消失。
